## 1) Content-Briefing (Ratgeber)
Beispiele: *EXT /// Hannoversche | riester rente kündigen | Content-Brief*, *EXT /// Hannoversche | altersvorsorge reformgesetz | Content-Brief* (Präfix „EXT ///" = an externe Texter/WortLiga)

**Aufbau (nummerierte Abschnitte):**
1. **Allgemeine Informationen** — Textlänge (z. B. „ca. 2.200 Wörter"), Thema, Format (Ratgeber / Produktseite / Blog), Tonalität, Sprachstil, **Ansprache („Direkt ‚Sie'")**, Zielgruppe (Bullet-Liste von Personas).
2. **Keywords & Meta Daten** — Hauptkeyword, lange Nebenkeyword-Liste, **SEO Title** + **SEO Meta Description**.
3. **Struktur** — Elemente auf der Seite (Listen, Fettungen, H2/H3, externe Links, CTA) + CTA-Beispiele.
4. **Content** — Kernfragen (Bullet-Liste), **Artikel-Gliederung H1/H2/H3** und häufig der **komplett ausformulierte Mustertext** + FAQs.
5. **Semantische Begriffe (WDF*IDF)** — Bullet-Liste.
6. **Ähnliche Wettbewerberseiten** zum Hauptkeyword (als Inspiration, mit URL).
7. **Ähnliche Fragen (PAA)**.
8. **Einbindung: Keywords/semantische Begriffe** — Regeln (H1, erste/letzte 100 Wörter, kein Stuffing).
9. **GEO** — Empfehlungen für KI-Suchmaschinen/AI Overviews (1 Absatz = 1 Aussage etc.).

**Mini-Skelett:**
```
Content-Briefing „<Keyword>"
1. Textlänge / Thema / Format / Tonalität / Ansprache (Sie) / Zielgruppe
2. Hauptkeyword · Nebenkeywords · SEO Title · Meta Description
3. Struktur (Listen, Fettungen, H2/H3, CTA-Beispiele)
4. Kernfragen → Gliederung H1/H2/H3 → Mustertext → FAQs
5. Semantische Begriffe (WDF*IDF)
6. Wettbewerberseiten   7. PAA   8. Einbindungsregeln   9. GEO
```

## 2) Schwellenkeyword-Optimierung
Beispiel: *Hannoversche-2020-08-17-Schwellenkeyword-Optimierung*

**Aufbau:**
1. Einleitung mit **Stand-Datum** + Logik („Keywords an der Schwelle zu Top-10").
2. **Nummerierte KW-Übersicht** mit aktueller Position.
3. Pro Keyword: **monatl. Suchvolumen** → **URL** → **„Verlauf des Keywords"** (seit wann Ranking, aktuelle Position, Screenshot-Verweis) → **„Empfehlung:"** mit konkreten Einbau-Stellen (Absatz/Leistungspunkt/neue H2), Ziel-Keyword **fett markiert** → Hinweis „beim nächsten Backlinkaufbau im Linktext berücksichtigen".
4. Rückfragen in [eckigen Klammern].

## 3) OnPage-Empfehlung (Einzelthema)
Beispiel: *HL | OnPage Empfehlung: Breadcrumb Navigation* (auch: *Einbindung der Bilder bei Wissenswert-Artikeln*, *Steuerung der Rankings der Rechner-Seiten*)

**Aufbau:**
1. Titel + **„Stand: TT.MM.JJJJ"**.
2. **Ist-Zustand** beschrieben + Abbildung/Quelle-URL.
3. **„Generelle Informationen … aus SEO-Sicht"** (Wirkprinzip erklärt).
4. **„Optimierungsempfehlung …"**.
5. **„Ziel:"** als prägnanter Soll-Satz.

## 4) Strategie-Konzept (groß)
Beispiel: *Konzept zur Integration von Versicherungsratgebern*

**Aufbau:**
1. **Ziel-Satz** oben + **Inhaltsverzeichnis mit Seitenzahlen**.
2. Argumentationskette: Warum sinnvoll? → Abgrenzung (Ratgeber vs. Produkt-LP, mit GSC-Belegen) → Zusammenfassung der Vorteile (E-A-T).
3. Voraussetzungen → Inhalte (beispielhaftes **Inhaltsverzeichnis mit KW + SV** je Kapitel) → Medien/stilistische Mittel → (SEO-)Funktionen.
4. **Landingpage-Aufbau** (Wireframe) + Nutzerführung (CTAs, interne Verlinkung).
5. **URL-Struktur** als Tabelle: **Wettbewerber-Benchmark (cosmosdirekt.de) → Ist hannoversche.de → Empfehlung** (Silo-Format).
6. **Ausblick / Wie geht es weiter?** (Verantwortlichkeiten EOM vs. HL).

---

## 5) Penalty-/Ursachen-Analyse
Beispiel: *HL | Ursachen Ranking-Verschlechterung*, *Hannoversche | Hannes Backlinks, RLV und BU*

**Aufbau:**
1. **Crawl-Link** (Screaming Frog) oben.
2. **Google Updates & SI** — SI-Verlauf je Core-/Spam-Update, je Verzeichnis (WissensWert/BU/RLV).
3. **Penalty-Hypothese** — algorithmisch (Verzeichnis-Ebene) vs. Keyword-Ebene, hypothesengeführt.
4. **Große Änderungen** (Navigation/strukturierte Daten/BFSG) als mögliche Auslöser.
5. **Prüfpunkte nach Google-Quality-Guidelines** (verborgener Text/Links, massengenerierter Content, Scraping) + offene Selbstkritik.
6. **Technische Probleme** (Screaming Frog: noindex/Canonicals/500er/H1/Sitemap).
7. **Ad-hoc-Mail-Entwürfe an Sven** (Du-Ton) als Anhang im selben Doc.

## 7) Jahres-/Projektübersicht (Reporting)
Beispiel: *Hannoversche - 2020-12 - Projektübersicht / Jahresreview* (Ordner `Reportings / Monitoring`, je Jahr ein Unterordner 2018–2025)

**Spalten (fix):** `Aufgabenbereich | Task | Optimierte URL | Dokument | Jira / Confluence | Erfolg? / Was ist daraus geworden?`
- Letzte Spalte enthält **Ranking-Deltas als Beleg** im Format „<keyword> Pos. X → Y" und Sistrix-/GSC-Links.
- Gruppierung über „Aufgabenbereich" (Ratgeber-Aufbau, Content-Vergleiche, Gegensteuerung Ranking-Verluste, Meta Daten, OnPage, Schwellenkeyword, Technik/Websitestruktur, Off-Page, Sonstiges).
- Begleitend: monatlicher **Leistungsnachweis** „Hannoversche - Zeiterfassung monatlich" (aus TOGGL); Reporting visuell über Data Studio/Looker.

## 8) Content-Vergleich (Spreadsheet)

- Vergleich der eigenen Produktseite gegen Wettbewerber-Seiten (Themen-/Keyword-Abdeckung), als Grundlage für Content-Nachschärfung; Erfolg über Sistrix-SEO-Changes-Links belegt.

## 9) Off-Page / Backlink-Docs
Beispiele: *Hannoversche | Hannes Backlinks, RLV und BU*, Disavow-/Defekte-Backlinks-Sheets, *advertising-adwords-keywords*
- **Linkbuilding-Brief:** Zielbudget, Kriterien (dofollow, SI/DR ≥ HL, Ziel-URLs RLV/BU, Zielgruppe), Tabellen-Vorgabe (Domain, SI, Preis, Beispiel-Link, Standzeit).
- **Disavow-Pflege:** laufende Updates der Disavow-Datei; Mietlink-Bewertung (ABAKUS).
- Off-Page wird zunehmend **GEO-begründet** (Nennung in autoritären Quellen für KI-Sichtbarkeit).
