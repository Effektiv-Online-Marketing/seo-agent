# SEO-Methodik — das EOM-Master-Template

EOM arbeitet mit **einer** standardisierten Output-Architektur, die pro Kunde nur befüllt wird. Beleg: identischer Aufbau über 6 Monate (NIBE), identische Keyword-Analyse-Vorlage NIBE **und** VHV, gemeinsamer Looker-Master.

Der SEO-„Audit" ist **kein klassisches Slide-Deck**, sondern **Looker-Dashboard + strukturiertes Monatsreport-Dokument**. Einziges echtes Deck: der Strategie-Scan fürs Jahresgespräch.

**Grundlogik überall:** Befund -> Begründung -> Maßnahme.

## Keyword-/Analyse-Deliverable (feste Kapitel, identisch NIBE <-> VHV)
1. Executive Summary / Key Takeaways
2. Phase 0 – Pre-Check: Rankt die Domain fürs Seed-Keyword? (Sistrix)
3. Domain-Kennzahlen (SI, organische KW, Seite 1/2, Backlinks) + Top-Wettbewerber (Keyword-Overlap %)
4. SERP-Analyse: PAA-Fragen, SERP-Features, Top-10-Domains, Intent-Verteilung
5. Keyword-Cluster + Suchvolumen (Tabelle, Datenquelle + Datum)
6. Wettbewerber-/Content-Benchmark
7. Kategorisierung BOFU / MOFU / TOFU
8. Content-Empfehlung (URL, Title ~57 Z., Struktur)

-> Kopf immer mit **Datum, Datenquelle, Analyst**; Einschränkungen offen markiert.

## Strategie-Scan / Jahresgespräch (7-Slide-Deck, produktisiert)
Strategie/Scan -> Neue Herausforderungen -> Leistungserweiterung (Strategie-Unit) -> Der Scan (3 Schritte) -> Ablauf (4 Steps) -> Nächster Schritt/Ansprechpartner -> Was „Strategie bei EOM" bedeutet.

## Prinzipien
- Nichts erfinden: kein Volumen/Ranking/Backlink schätzen. Fehlende Daten offen benennen.
- Neue/abgeschaltete Seiten aus der Wertung ausnehmen (Vergleichbarkeit).
- Externe Einflüsse offen benennen (Core Updates, num=100-Abschaltung etc.).
